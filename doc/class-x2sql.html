public x2sql
# where( mixed $set )

The where-filter can be a deep nested array, with three to five fields. Normally 3 fields are used to get a operand operator operand construct.

array("three","<>",3);

the "between" operation accepts 5 fields:

array(23,"between",8,"and",400);

The default handling is string escape if not a recognized operator. Use the HelperClasses to force a specific type: x2bool,x2number,x2keyword,x2func. And supqueries are supported , of course. <code> x2sql::query()->select()->from("table")->where(array( x2sql::query()->select("id")->from("table")->where(array("name","=","herbert")), "=",234 ))->command;
